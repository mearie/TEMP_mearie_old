PyFunge is an implementation of Befunge-93/Funge-98 interpreter in Python. Its goal is fully functional, compliant and optimizing implementation of Funge-98. Strikingly, such implementation is rare, according to Mycology test suite.

This project was started in 2004 at KLDP, forgotten for five years, and finally revived in 2009.

Development

See Mercurial repository for now. I'm targeting next major release, 0.5, to the following features:

  • Mostly (if not fully) comforming to Funge-98 specification.
  • Complete abstraction of Funge environment, easily pluggable API
  • Implementation of common fingerprints.

I also interested in the following subjects, though they will take much longer:

  • Integrated development environment for Funge-98. It will require 3D graphics, instruction flow visualization etc.
  • More fingerprints like Python binding.
  • Optimization and static compilation of Funge-98 program. This is a huge challenge, but nevertheless there is on-going JIT compilation attempt.
